**Critical finding — no per-borrower risk hook in Blend:**
Blend's risk parameters are strictly asset-level (CollateralFactor, LiabilityFactor per asset, same for all borrowers). There is no pre-borrow callback or external oracle slot that ZKredit can plug into. Blend's oracle slot is for price feeds only.

**What is feasible in 2 weeks — ZKreditGateway pattern:**
- Build a `ZKreditGateway` Soroban contract that:
  1. Reads `RiskAttestation::get_attestation(caller)` to get ZK-verified risk score
  2. Enforces a custom borrow cap / gating condition based on the score
  3. Forwards the `submit()` call to the Blend Pool via `blend-contract-sdk`
- The `blend-contract-sdk` Rust crate (v2.25.0) is published and usable via `contractimport!`
- Cross-contract calls into Blend are fully supported from Soroban

**Verdict — CONDITIONAL PASS:**
Blend testnet is live, the Rust SDK exists, and a gateway pattern demonstrating ZK-gated Blend access is feasible within 2 weeks. However, Blend's per-borrower collateral factor adjustment is NOT possible without forking Blend — risk enforcement lives in ZKreditGateway, not in Blend's own pool. 

**Fallback (if gateway scope is cut):** MockLendingPool only. Blend integration deferred to M2. The `ZKreditGateway` is now recorded as a Day 5-7 stretch goal.
