# Ishita Standup — 2026-07-03

## Wired the co-sign attestation seam (handoff-ishita-cosign-attestation.md)

Soham handed off two built-but-unwired seams (server `build_risc0_attestation_cosigned_xdr`,
client `submitCosignedAttestation`) plus one ML-side piece. Completed the wiring on
my side.

### Shipped today
- **New API route `POST /api/v1/attest/{addr}/prepare`** (`api/routes/v1.py`) — scores
  the wallet, then returns `AttestationPrepareResponse` = the scored fields **+**
  `partial_xdr` + `submission_mode` + `submission_detail`. This is what the
  frontend's `submitCosignedAttestation()` finishes signing in Freighter. The
  existing `POST /attest/{addr}` (submit → `tx_hash`) is unchanged.
- **Per-wallet RISC Zero prover driver** (`ml/risc0/prover.py`) — takes the
  distilled student's *selected transformed vector* (via
  `build_selected_vector_from_raw`, the same path `attest()` uses), writes it in
  the host's `ZKREDIT_FEATURE_VECTOR` JSON format, derives a 32-byte
  identity commitment from the wallet's decoded ed25519 key, and shells out to the
  host to produce a real per-wallet `seal`/`journal`. Gated on toolchain
  availability (`r0vm` + `cargo` + `docker`); raises `Risc0ProverUnavailableError`
  when absent.
- **Rust host** (`ml/risc0/host/src/main.rs`) — output dir is now overridable via
  `ZKREDIT_OUT_DIR` (default unchanged = the committed fixtures dir), so each
  attestation's receipt lands in its own temp dir.
- **`prepare_attestation_submission`** now takes optional live `seal`/`journal`:
  `live_cosign` when a real receipt is produced, else honest `demo_fixture_cosign`
  fallback to the committed fixture (Global Rule #2). The route degrades to the
  fixture when the prover is unavailable — it never 500s on missing toolchain.
- Tests: `ml/tests/test_risc0_prover.py` (input formatting, commitment, availability
  gate) + two `api/tests/test_routes.py` cases (fixture fallback + live-receipt
  pass-through). `ruff` + `mypy --strict` clean on `/api`; 15 tests green.
