## Context: KYC + Multi-Wallet Design

Plan approved for Days 3–10. Key architecture decisions locked:

1. **ZK identity circuit**: circom 2.0 + snarkjs (NOT EZKL — EZKL is ML-only). Poseidon preimage, ~500 constraints, <1s proof. Outputs Groth16 BN254 proof compatible with existing `verify_groth16`.
2. **Ed25519 wallet ownership**: Soroban's `wallet.require_auth()` handles it natively — zero circuit work.
3. **KYC**: Attestor-signed on-chain flag (`kyc_verified = true`) after off-chain Persona check. ZK part proves you OWN the identity commitment; attestor's tx signature proves KYC was done.
4. **Multi-wallet score sharing**: Any wallet in a group returns the group's best `risk_bucket` when queried. Wallet A's address never appears when querying Wallet B.

DG6 gate (Day 8 EOD): circom circuit compiles, snarkjs proof generates in <5s, VK registered on testnet via `register_verification_key`, proof verified by `verify_groth16`.
