- **Built the adversarial near-threshold parity harness.**
  Added boundary-focused tracing/probing to `ml/models/risc0_export.py`:
  - `trace_exported_forest(...)`
  - `trace_live_forest(...)`
  - `near_threshold_parity_report(...)`
  - supporting dataclasses for branch traces and mismatch logging
  What it does, step by step:
  1. start from **real raw population rows** so we only test nodes the model
     actually reaches in practice
  2. project those rows into the student's **selected transformed feature space**
     (because that is where the tree thresholds actually live)
  3. trace the exact split nodes visited by the exported forest
  4. for each visited node, generate adversarial copies where the deciding
     feature is set to:
     - `nextafter(threshold, -inf)`
     - `threshold`
     - `nextafter(threshold, +inf)`
     - `threshold ± 1e-9 * max(1, |threshold|)`
     - `threshold ± 1e-6 * max(1, |threshold|)`
  5. run each adversarial vector through:
     - the exported f64 artifact logic
     - the live sklearn student
  6. compare:
     - bucket
     - `confidence_bps`
     - branch direction at the **exact targeted split node**
  This is the test that actually probes the knife-edge cases generic real-row
  parity never hits.

- **Added a dedicated threshold-check CLI.**
  New script:
  - `scripts/check_risc0_threshold_parity.py`
  It runs the adversarial harness against the local model/population data with a
  row cap and a per-row node cap so the check is cheap enough to run repeatedly.

- **Adjusted the "live" branch trace to mirror sklearn's real inference path.**
  Important subtlety discovered while building the harness:
  - the exported artifact uses the intended f64 comparison contract
  - sklearn's live tree routing appears to compare using lower-precision
    (`float32`) inputs internally
  So `trace_live_forest(...)` now casts inputs the same way before comparing,
  otherwise the branch-trace diagnostic would falsely look cleaner than the real
  live model behavior.

- **Turned exact-tie behavior into a measured report field, not a hidden
  assumption.**
  The exported artifact follows the contract strictly: `feature <= threshold`
  goes left. For the live sklearn path, exact-threshold ties can fail to go left
  once the input is rounded into the live model's lower-precision route. Added:
  - `live_tie_nonleft_cases`
  to the near-threshold report so those cases are surfaced explicitly.
