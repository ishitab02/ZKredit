## 1. What this session did (one line)

Redesigned the `/attestation` page into a premium, calm, "high-end financial
credential" experience — deliberately **different from the landing page** —
wired to the real backend, and iterated to match an approved design mockup.

## 2. Design direction (locked with the user)

- Style: **Origin Financial–inspired** premium dark fintech. Elegant, trustworthy,
  calm — "revealing a credential", not a technical dashboard. Explicitly NOT the
  landing page's cinematic horizontal-scroll pipeline.
- Palette: **matches the landing** (deep teal-black ground, `teal-bright`
  #7FEBD9, pink/lavender haze) **plus a champagne/gold accent** (#E9CE9E /
  #F6E7C4 / #C6A667) for the luxe feel.
- Centerpiece: a **private-bank "metal card"** — brushed sheen, cursor-tracked
  specular sweep, EMV chip, embossed champagne score, and a **canvas-engraved
  guilloché seal**. It **flips** (click the card; cursor signals it) to reveal
  engraved on-chain provenance on the back.
- **Design guide / mockup (Claude artifact):**
  https://claude.ai/code/artifact/b9b095ad-e253-41a3-937d-59f3458a96dd
  Treat this as the visual source of truth. It is a static HTML mockup with
  placeholder data; the real React page mirrors it exactly.

## 3. Page structure (one route, `/attestation`, no tabs)

Left-aligned single column (`max-w-[1080px]`), calm corner-gradient background
(teal top-right, faint pink top-left, gold bottom):

1. **Hero** — gold "Attestation" eyebrow, "Your standing, / privately proven",
   one-line sub. Left-aligned.
2. **Connect panel** — landing-style `btn-primary` "Connect Freighter" (pink
   pointer-follow glow). After connect: wallet chip with a glowing "Testnet".
3. **Process** — "The process" kicker + "Four steps, then a credential" +
   `btn-ghost` "Request attestation". The **four steps (Read → Prove → Sign →
   Verify) are always visible once connected** — dimmed at rest, lighting up
   one-by-one as backend phases advance.
4. **Metal credential** — the flip card (front = score/bucket/confidence/seal +
   proof hash/attestor/expires/Stellar Expert link; back = distilled + full
   model hash, issued date, magnetic-stripe motif).
5. **Receipts** (side by side, both open by default) — "What the model saw"
   (real SHAP `top_features`) and "Model & proof system" (`/model-info`).
6. **Footer** — the shared landing `Footer` (same on both routes).

## 5. Backend wiring — what's real vs pending

- **Real, no fakes**: score, risk bucket, confidence, SHAP features (all from the
  `/api/v1/attest/{addr}/prepare` response, which extends `AttestationResponse`
  server-side); proof hash / attestor / model hashes / issued / expiry (on-chain
  read-back via `getAttestation`); proof system + zk capability (`/model-info`);
  Stellar Expert link (real `txHash`).
- The full attestation flow is: `POST /auth/session` → `POST
  /attest/{addr}/prepare` → wallet co-signs (Freighter) → submit to Soroban →
  read back → render card. State machine + all error kinds are in
  `OnChainAttest.tsx`.
- **Pending / needs backend to see the card render locally**: a CORS-reachable
  `VITE_API_URL` and Freighter on Testnet with a funded account. The Fly backend
  still blocks the Vercel origin at preflight (CORS) per `docs/v2_plan.md`.
- **No fabricated preview** — the credential only renders after a real
  attestation completes (a dev sample-data preview was added then removed at the
  user's request).

## 6. Fonts / typography note

The attestation page intentionally uses the **artifact's neutral system font**
(the artifact sandbox can't load Clash Display/Satoshi, so it falls back to
`system-ui`; the user wanted that look). Scoped via `.attest-artifact-font` on
the attestation `<section>` + an **un-layered** CSS rule in `index.css` (so it
beats Tailwind's `font-display`/`font-sans` utilities). The **landing, nav
wordmark, and footer keep Clash Display** — do not globalize the override.
