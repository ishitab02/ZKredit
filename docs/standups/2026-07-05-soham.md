# Soham Standup — 2026-07-05

## Proof-gated identity linking — end to end

Now that DG6 passed, the multi-wallet feature is real ZK, not a stand-in:
linking a wallet requires an on-chain-verified proof that you know the identity
secret.

### Contracts (committed 43bf904)
- Moved the Groth16 verifier to the `shared` crate (used by both RiskAttestation
  and WalletIdentity); added `groth16::nth_public_input`.
- `WalletIdentity`: admin `set_identity_vk`; `register_wallet(wallet, commitment,
  proof_bytes)` verifies the proof against the identity VK and binds its public
  input == commitment. Optimistic when no VK is set.
- e2e-tests `--features dg6`: valid proof registers; commitment-mismatch and
  tampered proof rejected. deploy-testnet.sh registers the identity VK.

### Frontend (this commit)
- `lib/zk/identity-proof.ts`: in-browser proving — `proveIdentity()` runs
  snarkjs `groth16.fullProve` against `/zk/identity.{wasm,zkey}` and serializes
  the proof to the Soroban blob format (same encoding as `convert.js`). Commitment
  comes from the proof's public signal.
- Shipped circuit assets to `frontend/public/zk/` (wasm 1.6MB, zkey 182KB;
  fetched on demand). Added snarkjs; snarkjs is a lazy dynamic-import chunk.
- `registerWallet(wallet, commitment, proofBytes)`; `Identity.tsx` now generates
  a real Poseidon commitment + proof and links wallets against the proof-gated
  contract.
- Updated `docs/demo.md` (real Poseidon + on-chain proof verification; DG6 done).

### Verification
- `cargo` workspace green; e2e `--features dg6` (6) green.
- Frontend `tsc -b` + `vite build` clean.
- Node sanity: a fresh proof from the shipped `/zk/` assets verifies
  (`snarkjs.groth16.verify` = true); blob conversion is byte-identical to the
  DG6-proven `convert.js`. Browser runtime of the link flow is the one piece
  only checkable in-browser (manual).
